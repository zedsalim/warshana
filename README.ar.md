## المميزات

- **روايتان:** التبديل الفوري بين ورش (رواية ورش عن نافع) وحفص (رواية حفص عن عاصم) — انقر على الشارة في الرأسية أو استخدم محدِّد الرواية في الشريط الجانبي
- التنقل بالسورة أو الجزء أو الصفحة أو الآية من الشريط الجانبي
- تشغيل الصوت آية بآية مع قراء متعددين لكل رواية
- أوضاع التشغيل: آية، صفحة، سورة، أو جزء
- التحكم في سرعة التشغيل (0.5×–2×) وتكرار التلاوة (بما في ذلك التكرار اللانهائي)
- ضبط حجم خط نص القرآن الكريم
- تمييز الآية النشطة مع التمرير التلقائي
- تبديل وضع القراءة بملء الشاشة
- حفظ الإعدادات عبر `localStorage` بين الجلسات

## تشغيل التطبيق

يمكنك استخدام التطبيق بثلاث طرق:

- **أونلاين:** قم بزيارة [sarayan.pages.dev](https://sarayan.pages.dev/) مباشرة من المتصفح.
- **أندرويد:** نزّل وثبّت ملف APK من [Releases](https://github.com/zedsalim/sarayan/releases) _(قم بتفعيل "التثبيت من مصادر غير معروفة" إذا طُلب منك ذلك)_.
- **محلياً:** استنسخ المستودع وشغّله بنفسك (مطلوب للصوت في وضع أوفلاين).

### 1. الحصول على الكود

```bash
git clone https://github.com/zedsalim/sarayan
cd sarayan
```

### 2. تشغيل الخادم المحلي

يجب تشغيل التطبيق عبر HTTP — فتح `index.html` مباشرةً لن يعمل. اختر أياً من الطرق التالية:

```bash
# Python 3
python -m http.server 3000

# Python 2
python -m SimpleHTTPServer 3000

# Node.js (npx)
npx serve .

# VS Code
# ثبّت امتداد "Live Server" وانقر على "Go Live"
```

ثم افتح [http://localhost:3000](http://localhost:3000) في متصفحك.

## إعداد الصوت أوفلاين

يقوم التطبيق افتراضياً ببث الصوت من روابط أونلاين. لاستخدامه **بشكل كامل أوفلاين**، نزّل ملفات الصوت وضعها محلياً في المجلد الفرعي الصحيح للرواية:

**1. تنزيل ملفات الصوت**

نزّل مجلد القارئ من Google Drive:
[📁 تنزيل الصوت — Google Drive](https://drive.google.com/drive/folders/10Iujd_rNoflXmSNfwkyPmmq9aOCLSejO?usp=drive_link)

**2. وضعها في المجلد الصحيح**

يتم تنظيم الصوت حسب الرواية:

```
audio_local/
├── warsh/
│   └── reciter_name/   ← مجلد قارئ ورش
│       ├── 001/
│       │   ├── 001.mp3
│       │   ├── 002.mp3
│       │   └── ...
│       ├── 002/
│       └── ...
└── hafs/
    └── reciter_name/      ← مجلد قارئ حفص
        ├── 001/
        │   ├── 001.mp3
        │   └── ...
        └── ...
```

> ⚠️ يجب أن تتطابق أسماء المجلدات تماماً مع مفاتيح القارئ المحددة في `RIWAYA_CONFIG` داخل `script.js`.

**3. شغّل الخادم المحلي واستمتع بالتلاوة أوفلاين**

يكتشف التطبيق تلقائياً ما إذا كانت ملفات الصوت المحلية موجودة (عبر طلب HEAD واحد لكل سورة) ويستخدمها. وإذا لم تكن موجودة، فإنه يتراجع إلى البث أونلاين دون الحاجة إلى أي إعداد.

## بيانات الصوت

يُقسَّم الصوت الأونلاين إلى **ملفات مجزّأة لكل قارئ ولكل سورة** لتحميلها عند الطلب، يتم توليدها بواسطة `scripts/split_reciters.py`:

```
assets/audio_cloud/
├── warsh/
│   └── lazy/
│       └── reciter_name/
│           ├── index.json   ← قائمة السور المتاحة
│           ├── 001.json
│           ├── 002.json
│           └── ...
└── hafs/
    └── lazy/
        └── reciter_name/
            ├── index.json
            ├── 001.json
            └── ...
```

تحتوي كل ملف سورة على أزواج account/token من شبكة CDN يستخدمها التطبيق لإعادة بناء روابط الصوت عند التشغيل. لإعادة التوليد:

```bash
python split_reciters.py --scan assets/audio_cloud
```

## اختصارات لوحة المفاتيح

| المفتاح      | الإجراء                |
| ------------ | ---------------------- |
| `Space`      | تشغيل / إيقاف مؤقت     |
| `ArrowLeft`  | الآية التالية          |
| `ArrowRight` | الآية السابقة          |
| `0–9`        | الانتقال من 0% إلى 90% |
| `+` / `-`    | زيادة / تقليل السرعة   |
